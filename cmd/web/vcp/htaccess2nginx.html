htaccess 转 Nginx 重写规则

把 Apache 的 .htaccess 伪静态与重写规则翻译成 Nginx 的 rewrite 配置，转换在浏览器本地完成，内容不会上传。

复制

从 Apache 迁移到 Nginx

Apache 用 .htaccess 里的 RewriteRule、RewriteCond 做伪静态和重定向，Nginx 没有等价机制，需要改写成 location 块里的 rewrite 指令。这个工具帮你完成大部分机械转换：把常见的重写、重定向、条件判断译成对应的 Nginx 语法。

转换结果建议作为草稿核对，不要直接照搬上线。.htaccess 是按目录生效的，Nginx 的 rewrite 写在 server 或 location 上下文里，两者的匹配顺序和作用范围并不完全一致，复杂规则（如基于环境变量、Header 的条件）往往需要手工调整。
