JS 代码混淆

把 JavaScript 代码改写成难以阅读的等价形式，在不改变运行结果的前提下提高被直接照抄的门槛。处理在浏览器本地完成。

复制

混淆不是加密

这里做的是混淆而非加密：通过变量改名、编码字符串、打乱结构等手段，把代码变成人难以读懂、但浏览器仍能正常执行的形式。它不需要密钥，也不能真正解密——混淆后的代码依然会被运行，因此理论上都可以被还原或调试出来。

它的作用是提高阅读和直接照抄的成本，适合给前端脚本做一层基础保护，但不要把任何密钥、密码等机密写进前端代码指望靠它保密。粘贴代码后点按钮即可得到混淆结果。
